Ball Balancer

Abstract

Introduction

The goal of this project is to balance a steel ball bearing on a flat plate which is an inherently unstable system, since any tilt causes the ball to roll freely in that direction. The robot achieves this by using a pressure-sensing plate to track the ball's position, and three stepper motors to tilt the plate in response, correcting the ball's trajectory as it moves off center. This project served both as my 16-299 final project and as a fun build and has since earned a permanent spot as a desk ornament.

The control of the plate is built around a PID controller, adapted from an existing file you can find at https://www.instructables.com/Ball-Balancer/, which also included several target patterns for the ball to follow. Getting the PID tuned to work consistently across all patterns was the central challenge, as when the ball needed to both correct its position and track a moving target simultaneously, it frequently overshot and fell off the plate. To address this, I tuned the controller extensively and introduced a feed-forward term to smooth the plate's motion. I also explored an LQR controller as an alternative approach. The primary benchmarks for success are how long the ball remains on the plate, and how accurately it tracks its target position.

Sensing

The pressure plate I used reads out x and y coordinates (x, y) of where it senses the greatest pressure. One of the corners is marked as (0, 0), and the opposing corners reads a maximum value of approximately (1000, 1000). To get proper readings from the plate, I put pressure as centered as possible and put that as an offset (which came out to (500,500)) within my PID controller. I subtracted this from all measurements, which put the center as the origin, and the other positions are diplsacements from that point in sensor unites. The derivative term in the PID then approximates the ball's belocity by computing the change in error between each loop iteration which is run every 20 milliseconds. There isn't an explicit conversion to actual distance in the real-world, just sensor units along with PID gains.

PID, LQR, and Feed-Forward Control

The robot runs the Arduino program using a Teensy 4.1 Microcontroller, where the 3 stepper motor A,B, and C, as well as the pressure plate are connected. The robot is classified as a 3RPS parallel manipulator, which essentially means that hte three legs push on the plate from below by changing their heights independent of each other, giving the plate a wider range of motion. There is an included library that handles the inverse kinematics of turning the desired tilt (x, y) into the actual angle for the 3 motors and their links. Within the control loop, this is what is repeated every 20 milliseconds:
  • Read the touchscreen's output in order to get the balls (x, y) position
  • Subtract the offset to get the error or distance from center
  • Run the PID controller to compute how to tilt the plate as well as what direction to tilt
  • Put hte tilt command through the inverse kinematics and that gives motor commands to achieve the tilt
  • Move the stepper motors towards the calculated position
